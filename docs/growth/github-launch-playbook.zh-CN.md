# BeeAGI GitHub 开源启动作战手册（4 周）

目标：把项目从“可用”推进到“可发现、可传播、可贡献”。

## 北极星指标

- `Star` 增长速度（每周）
- `Watch` 数量
- `Issue` 与 `PR` 的有效互动数
- 外部来源流量（社媒/社区）到仓库访问

## 第 1 周：打磨首页转化

1. README 首屏三件套：
   - 一句话价值主张
   - 30 秒演示入口
   - 快速启动命令
2. 补 Topics（最多 20 个，精准即可）：
   - `multi-agent`
   - `llm-orchestration`
   - `tauri`
   - `fastapi`
   - `ai-workflow`
   - `private-deployment`
3. 创建 `v0.2.0` Release，附中英文说明和演示脚本。

## 第 2 周：做“可复现”的证据

1. 发布 3 个可复现场景 Demo：
   - 编程场景
   - 办公场景
   - 研究分析场景
2. 每个 Demo 都包含：
   - 输入样例
   - 预期输出
   - 反馈后变化点
3. 增加一页“能力边界说明”，明确适合/不适合场景。

## 第 3 周：激活社区协作

1. Discussions 开启三个固定栏目：
   - `Q&A`
   - `Roadmap`
   - `Show and Tell`
2. 每周固定处理：
   - 新 Issue 在 24 小时内响应
   - 标记 `good first issue`（至少 3 个）
3. 发起一次“方向投票”讨论（例如：下个重点场景）。

## 第 4 周：内容分发与复盘

1. 统一发布节奏（每周至少 2 条）：
   - X / 知乎 / 掘金 / V2EX / Reddit
2. 内容结构固定：
   - 一个真实问题
   - BeeAGI 如何处理
   - 可复现链接
3. 周复盘：
   - 流量来源
   - Star 转化率
   - 哪类内容带来最多有效讨论

## 内容模板（可直接复制）

### 发布帖模板

- 标题：`我们把用户反馈做成了技能实时进化闭环（可回滚）`
- 三段正文：
  1. 解决什么问题
  2. 怎么验证有效
  3. 仓库链接 + 30 秒演示链接

### 更新日志模板

- 新增了什么
- 修复了什么
- 下一步要做什么

## 执行原则

1. 永远先给“可运行演示”，再讲架构。
2. 永远先给“可复现结果”，再谈愿景。
3. 永远保持“快速响应 + 稳定节奏”。
