# 30 秒演示脚本（中文）

## 演示目标

展示 BeeAGI 如何从“任务输入 -> 交付结果 -> 反馈进化”走完整闭环。

## 演示前准备

- 后端已启动：`http://127.0.0.1:8000`
- 桌面端已启动
- 打开工作台页面（非 LLM 页面）

## 直播脚本

### 0-10 秒：输入任务

1. 选择场景：`编程`
2. 任务目标：
   - `实现用户状态筛选 + 分页，并补齐单元测试`
3. 上下文引用：
   - `repo://frontend/src/pages/users,repo://backend/app/api/routes/users.py`
4. 约束 JSON：

```json
{
  "doneWhen": ["tests pass", "lint clean", "api backward compatible"],
  "language": "zh-CN"
}
```

### 10-20 秒：执行流程

1. 点击 `一键跑完整流程`
2. 同时讲解：
   - “Scout 先侦察上下文”
   - “Worker 规划并执行”
   - “主区域出现可交付结果”

### 20-30 秒：反馈与进化

1. 点击 `立即自动反馈`（或手工反馈）。
2. 打开右侧 `进化` 页签。
3. 展示：
   - 蜂群事件更新
   - Scout 信息素列表
   - 可选点击 `运行巡检`

## 收尾话术

“BeeAGI 不只是完成任务，还能基于真实用户反馈进行受控自进化，并且每次变化都可审计、可回滚。”
