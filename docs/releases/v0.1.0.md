# BeeAGI v0.1.0

Release date: 2026-05-17

## Highlights

- Swarm architecture baseline is available end-to-end:
  Scout -> Worker -> Worm -> Queen.
- Desktop experience now focuses on user outcomes:
  chat-style timeline + deliverable-first workspace.
- Evolution controls are production-oriented:
  shadow replay, canary routing, promote, rollback, audit trail.
- LLM operations are now visible and configurable:
  runtime config page and token usage statistics.

## Core APIs

- Tasks: `POST /tasks`, `GET /tasks/{id}`
- Feedback: `POST /tasks/{id}/feedback`, `POST /tasks/{id}/auto-feedback`
- Skills/Evolution:
  - `POST /skills/{id}/candidate`
  - `POST /skills/{id}/promote`
  - `POST /skills/{id}/rollback`
  - `POST /skills/{id}/candidate/{candidate_id}/shadow-replay`
  - `GET /skills/{id}/candidate/{candidate_id}/canary-status`
  - `POST /evolution/auto-promote`
  - `GET /evolution/events`
  - `GET /evolution/candidate-audits`
  - `GET /evolution/hardening-report`
- LLM:
  - `GET /llm/config`
  - `PUT /llm/config`
  - `GET /llm/token-stats`
